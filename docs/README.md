# Documentation

Durable project documentation lives here. Keep this directory small at first, then add documents when they help future work.

## Index

- `CONVENTIONS.md`: repository-wide path, naming, generated-file, and documentation rules.
- `TRAINING_RUNTIME.md`: runtime boundary for GPU training versus platform inference.
- `adr/`: optional architecture decision records for decisions that are hard to reverse or easy to forget.

## Guidelines

- Start each durable document with a short purpose statement.
- Use paths relative to the repository root.
- Prefer links to source files over copied code.
- Update this index when adding or removing durable docs.
