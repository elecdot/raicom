# Architecture Decision Records

Use this directory for decisions that are hard to reverse, surprising, or based on a real tradeoff. Do not create ADRs for routine implementation choices.

Suggested filename:

```text
0001-short-title.md
```

Use the next sequential number when adding a new ADR.

Suggested shape:

```md
# <Decision Title>

One to three sentences describing the context, decision, and reason.
```
