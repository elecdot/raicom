# Conventions

This file holds repository-wide conventions that are stable enough to preserve across projects.

## Documentation

- Keep root README content focused on orientation and navigation.
- Put subtree-specific rules in that subtree's README when the directory needs
an entry point, index, rules, or conventions to stay readable and maintainable.
- Do not add a local README for a simple directory that has no durable
maintenance needs beyond its contained files.
- Update docs in the same change as the behavior or workflow they describe.
